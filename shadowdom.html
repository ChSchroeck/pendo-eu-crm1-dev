Open Shadow DOM
Web components with mode: 'open'
Shadow hosts are in the light DOM. host.shadowRoot returns the root — Pendo can traverse it.
ℹ
Enable shadow DOM traversal in your Pendo agent. With that on, Pendo walks .shadowRoot on each host and can tag these elements. The shadow host below has data-pendo-id for reliable targeting.

Date picker component (open shadow root)

#shadow-root (open) — contents rendered by JS below

The fields inside are attached to the shadow root of #open-datepicker-host. In DevTools, expand the host to see the shadow tree. With Pendo shadow DOM support on, these are taggable.

Closed Shadow DOM
Components with mode: 'closed'
host.shadowRoot returns null. Inner elements are inaccessible without early agent interception.
✕
Standard Pendo tagging cannot reach inside. The host is visible in the light DOM; the internals return null. Your options: (1) load the Pendo agent before these components mount to intercept attachShadow(), (2) tag the shadow host itself, or (3) use pendo.track() from inside the component code.

Video player component (closed shadow root)

#shadow-root (closed) — wall is up

The play/pause, volume, and progress controls inside are attached to a closed shadow root. document.querySelector('#closed-player-host').shadowRoot returns null — try it in your DevTools console.

Payment widget (closed shadow root — like Stripe Elements)

Track events from inside closed components with pendo.track()

// Inside your closed web component's source code const shadow = element.attachShadow({ mode: 'closed' }); // Wire up events where you have direct access shadow.querySelector('.play-btn').addEventListener('click', () => { pendo.track('Video Played', { componentId: 'closed-player', source: 'shadow-dom' }); }); shadow.querySelector('.pay-btn').addEventListener('click', () => { pendo.track('Payment Initiated', { method: 'card', source: 'shadow-dom' }); });
